# motogarage_parser

**Назва проекту:** motogarage_parser
**Опис:** Парсер та інтерпретатор для MotoGarage DSL, мови для керування колекціями мотоциклів.

---

## План

Цей проект реалізує парсер для кастомної Доменно-Специфічної Мови (DSL) для керування "гаражем" мотоциклів.

### 1. Граматика

Парсер призначений для обробки текстових файлів (наприклад, `.moto`), які містять набір команд. У фінальній версії, мова буде підтримувати:

* Визначення нових байків (`DEFINE bike ... { ... }`)
* Запити до колекції (`GET BIKES WHERE ...`)

На поточному, початковому етапі, граматика (`src/grammar.pest`) визначає лише базові "атоми", такі як **ідентифікатори** (`ident`).

### 2. Процес

1.  **Pest Parser:** Я  використовуємо бібліотеку `pest` для визначення нашої граматики у форматі PEG (Parsing Expression Grammars) у файлі `src/grammar.pest`.
2.  **Генерація парсера:** `pest_derive` автоматично генерує парсер (`MotogarageParser` у `src/lib.rs`) на основі цієї граматики.
3.  **AST (Абстрактне Синтаксичне Дерево):** (Буде додано пізніше ) Згенероване "дерево пар" (Pairs) буде трансформовано у строго типізовані структури Rust (наприклад, `Command`, `Motorcycle`), які формують AST.

### 3. Як використовуються результати

(Буде додано пізніше ) Після того, як вхідний текст буде перетворено на AST, "інтерпретатор" (наприклад, структура `Garage`) буде обходити це дерево та виконувати команди:

* `DEFINE` буде додавати нові об'єкти `Motorcycle` у внутрішню колекцію.
* `GET` буде фільтрувати цю колекцію і повертати результати.
